NAME

log - configure logging output from other filter-sets  

SYNOPSIS

filterset log
{
    filename "bugle.log"
    flush "no"
    format "%f.%e: %m"
}
 

DESCRIPTION

The log filter-set does not do any logging itself; instead, it provides logging services to other filter-sets. It is automatically loaded when needed, but explicitly loading it allows options to be set.  

OPTIONS

filename
This sets the filename for logging. If not filename is given, the log is written to standard error.
flush
If this is enabled (it is not by default) then the log is flushed to disk after every entry. This may be required to obtain a complete log if the program crashes.
format
This optional string overrides the default log format. The following printf-style escapes are recognised:
%f
filterset
%e
event
%m
log message
%p
process ID
%t
thread ID (from pthread_self(3))
%%
a literal %
